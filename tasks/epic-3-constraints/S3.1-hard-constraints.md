# S3.1 — Hard constraints + feasibility checker

- **Epic:** 3 — Constraint modeling
- **Story points:** 5
- **Status:** ☑ Done
- **Depends on:** S1.2, S1.3

## Goal
Encode and enforce constraints that must never be violated.

## Tasks
- [ ] Fixed group sizes (or size range) per config
- [ ] Unique assignment: each participant in exactly one group
- [ ] All participants assigned (no orphans)
- [ ] `is_feasible(assignment, config) -> (bool, list[violations])`
- [ ] Helper to build a feasible random starting assignment

## Acceptance criteria
- Returns precise, human-readable violation messages.
- Detects under/over-full groups and double/zero assignment.
- Unit tests for each violation type.

## Notes
Both optimizers (local search, ILP) rely on this — local search rejects infeasible swaps,
ILP encodes these as hard constraints.
