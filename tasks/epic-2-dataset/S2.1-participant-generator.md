# S2.1 — Participant generator

- **Epic:** 2 — Synthetic dataset generation
- **Story points:** 5
- **Status:** ☑ Done
- **Depends on:** S1.2, S1.3

## Goal
Generate reproducible synthetic participants with tunable distributions for skills,
experience, and diversity attributes.

## Tasks
- [ ] Seeded RNG (`numpy.random.default_rng(seed)`)
- [ ] Skills: configurable number of skills, per-skill distribution (e.g., normal/uniform)
- [ ] Experience: ordinal levels with configurable proportions
- [ ] Diversity attributes: categorical (e.g., nationality, gender) with configurable mix
- [ ] Optional preferences: random likes/dislikes between participants
- [ ] `generate(n, config) -> list[Participant]`

## Acceptance criteria
- Same seed → identical dataset (regression test asserts this).
- Distribution parameters observably change the output.
- No duplicate participant ids.

## Notes
Document each distribution choice — it matters for the ethics discussion (S8.1) about
how synthetic data can bake in bias.
