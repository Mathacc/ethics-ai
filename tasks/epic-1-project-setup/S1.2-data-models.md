# S1.2 — Participant & Group models

- **Epic:** 1 — Project setup & data model
- **Story points:** 3
- **Status:** ☐ Not started
- **Depends on:** S1.1

## Goal
Define the core domain types used everywhere: participants, groups, and an assignment.

## Tasks
- [ ] `Participant`: `id`, `skills` (dict or vector), `experience` (ordinal level),
      `diversity_attrs` (e.g., gender, nationality), `preferences` (optional likes/dislikes)
- [ ] `Group`: `id`, `members: list[Participant]`, capacity
- [ ] `Assignment`: mapping participant_id → group_id, plus helper views
      (`groups()`, `group_of(pid)`)
- [ ] Validation (pydantic or `__post_init__`): non-empty id, skill values in range
- [ ] Round-trip helpers: to/from dict for serialization

## Acceptance criteria
- Models are immutable where sensible and validate bad input.
- `Assignment` can reconstruct full `Group` objects from a participant list.
- Unit tests cover validation and round-trip.

## Notes
Represent skills as a fixed-order vector internally for fast metric computation;
keep a human-readable name mapping for explanations (S7.x).
