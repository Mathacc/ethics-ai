# S1.3 — Config schema

- **Epic:** 1 — Project setup & data model
- **Story points:** 3
- **Status:** ☐ Not started
- **Depends on:** S1.2

## Goal
A single, validated configuration object that drives generation, constraints, optimizers,
and fairness weights — loadable from YAML/JSON.

## Tasks
- [ ] `Config` model: `n_groups` or `group_size`, hard-constraint toggles
- [ ] Soft-constraint weights: `w_skill_balance`, `w_diversity`, `w_preference`
- [ ] Optimizer params: max iterations, SA temperature schedule, time limit (ILP)
- [ ] Global `seed`
- [ ] `Config.load(path)` for YAML/JSON + sensible defaults
- [ ] Ship an example `config.yaml`

## Acceptance criteria
- Invalid configs (negative sizes, weights that don't make sense) raise clear errors.
- Defaults allow running the pipeline with zero config.
- Example config documented in README.
